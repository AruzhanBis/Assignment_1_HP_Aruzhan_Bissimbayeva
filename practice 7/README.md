# Практическая работа №7: Редукция и сканирование на GPU

## Цель работы
Изучение и реализация параллельных алгоритмов редукции и сканирования (префиксной суммы) на GPU с использованием CUDA. Проведение анализа производительности и оптимизация кода.

## Реализованные задачи

### Основные задачи

1. **Редукция суммы элементов массива на GPU** (`pr7_task1.cu`)
   - Реализовано ядро CUDA с использованием разделяемой памяти для оптимизации доступа.
   - Проверка корректности работы на тестовом массиве.
   - Вывод результата суммы элементов на экран.
   - Возможность запуска на Google Colab или локально через терминал с командой:
     ```bash
     !nvcc pr7_task1.cu -o pr7_task1 -arch=sm_75 -std=c++11
     !./pr7_task1
     ```

2. **Префиксная сумма (scan) на GPU** (`pr7_task2.cu`)
   - Реализовано ядро CUDA для вычисления префиксной суммы с использованием shared memory.
   - Проверка корректности работы на тестовом массиве.
   - Вывод первых элементов результата для визуальной проверки.
   - Запуск аналогично через nvcc:
     ```bash
     !nvcc pr7_task2.cu -o pr7_task2 -arch=sm_75 -std=c++11
     !./pr7_task2
     ```

3. **Анализ производительности** (`pr7_task3.cu`)
   - Замер времени выполнения редукции на GPU для разных размеров блока (128, 256, 512, 1024).
   - Сравнение с CPU-реализацией.
   - Вывод времени выполнения для каждого размера блока.
   - Запуск:
     ```bash
     !nvcc pr7_task3.cu -o pr7_task3 -arch=sm_75 -std=c++11
     !./pr7_task3
     ```

### Дополнительные задания

1. **Редукция минимума и максимума** (`dop_task1.cu`)
   - Реализована редукция для вычисления минимального и максимального значения массива.
   - Результаты выводятся на экран для наглядной проверки:
     ```
     GPU Reduction Result:
     Minimum value = ...
     Maximum value = ...
     ```
   - Запуск:
     ```bash
     !nvcc dop_task1.cu -o dop_task1 -arch=sm_75 -std=c++11
     !./dop_task1
     ```

2. **Алгоритм Blelloch Scan** (`dop_task2.cu`)
   - Реализован более эффективный вариант префиксной суммы.
   - Используется shared memory и up-sweep/down-sweep подход.
   - Вывод первых 10 элементов результата для проверки.
   - Запуск:
     ```bash
     !nvcc dop_task2.cu -o dop_task2 -arch=sm_75 -std=c++11
     !./dop_task2
     ```

3. **Исследование влияния размера блока на производительность** (`dop_task3.cu`)
   - Замер времени выполнения редукции на GPU при разных размерах блока (128, 256, 512, 1024).
   - Вывод времени для каждого размера блока.
   - Запуск:
     ```bash
     !nvcc dop_task3.cu -o dop_task3 -arch=sm_75 -std=c++11
     !./dop_task3
     ```

## Результаты работы
- Все задачи успешно реализованы и проверены на тестовых массивах:
  - 1024 случайных числа
  - 1 000 000 случайных чисел
  - 10 000 000 случайных чисел
- Корректно работают как сумма, так и префиксная сумма.
- Замеры времени показали значительное ускорение на GPU по сравнению с CPU.
- Дополнительные задания выполнены с наглядным выводом минимальных, максимальных значений и результатов Blelloch Scan.

## Запуск на Google Colab
- Все файлы `.cu` можно компилировать и запускать внутри Colab с использованием команд `!nvcc` и `!./имя_файла`.
- Для каждого задания достаточно заменить имя файла в командах компиляции и запуска.

## Выводы
- GPU обеспечивает значительное ускорение вычислений при больших массивах.
- Shared memory существенно повышает производительность редукции и сканирования.
- Оптимизация размера блока влияет на скорость выполнения.
- Применение алгоритма Blelloch Scan позволяет эффективно реализовать префиксную сумму на GPU.
